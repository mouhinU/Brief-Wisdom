# 系统开发 Skill

## 作用

用于 Brief-Wisdom 的系统级开发工作，包括后端模块、认证、权限、共享服务以及部署相关变更。

## 何时使用

当任务涉及跨模块功能、共享服务变更、访问控制调整或部署相关修改时使用。

## 核心规则

- 先阅读 AGENTS.md，并查看相关模块结构
- 优先做小而聚焦的修改，避免大范围重写
- 控制器保持轻量，业务流转交给服务层
- API 使用 DTO，并避免直接暴露持久化实体
- 遵循模块边界：web -> service/api -> persistence -> common
- 避免魔法值和硬编码的环境相关配置

## 推荐流程

1. 确定受影响的模块和相关层
2. 复用现有的服务、DTO、异常和返回结果模式
3. 在正确的层完成改动，并尽量减少影响面
4. 补充或更新相关测试
5. 使用对应的构建或测试命令进行验证

## 触发关键词

系统开发、架构变更、模块重构、跨模块功能、部署更新

## 优先级

涉及安全、权限、共享服务或部署相关变化时优先级较高；常规功能开发为中等。

## 默认输出要求

总结变更方案、列出受影响模块，并说明应执行的验证命令。

## 适用对象

从事后端、系统、AI 或部署相关工作的开发者。

## 避免事项

不要绕过现有安全检查、引入重复模式，或在未确认相关模块前进行跨仓库大改。

## 示例提示词

为用户系统新增一个需要权限控制的接口，并遵循 Brief-Wisdom 的后端开发规范。

# 后端架构 Skill

## 作用

用于 Brief-Wisdom 的核心后端变更，如接口、服务、仓储、DTO 以及模块边界调整。

## 何时使用

适用于新增接口、共享服务修改、仓储逻辑更新或跨层后端重构。

## 核心规则

- 遵循模块依赖方向：web -> service/api -> persistence -> common
- 控制器保持轻量，业务流转交给服务层
- 使用服务接口，并在对应 impl 包中实现
- 优先使用 DTO，避免直接暴露持久化实体
- 复用现有异常、返回结果和缓存模式

## 推荐流程

1. 确定目标模块和受影响层
2. 复用现有接口、DTO 和返回结果模式
3. 在正确层完成修改
4. 为受影响流程补充或更新测试
5. 验证对应模块的构建或测试

## 触发关键词

后端架构、服务层、仓储变更、DTO 更新、模块边界

## 优先级

涉及共享基础设施或跨模块行为时优先级较高；面向单一功能的后端改动为中等。

## 默认输出要求

说明受影响层、影响模块以及后续验证步骤。

## 适用对象

后端开发者和维护人员。

## 避免事项

当现有服务或 DTO 已足够满足需求时，不要引入新的模式。

## 示例提示词

按照 Brief-Wisdom 的服务层架构重构这个后端流程，并保持模块边界不被打破。

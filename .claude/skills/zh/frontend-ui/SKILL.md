# 前端 UI Skill

## 作用

用于前端界面开发、交互流程、页面状态管理以及与后端接口的集成。

## 何时使用

当任务涉及页面展示、交互逻辑、表单提交、状态管理或前端接口联调时使用。

## 核心规则

- 前端改动保持简洁，避免把复杂业务逻辑压入视图层
- 复用当前项目的组件和接口调用模式
- 对用户输入做必要校验，并处理加载、错误和空状态
- 避免将敏感逻辑或权限规则完全依赖前端

## 推荐流程

1. 先查看现有页面和组件结构
2. 复用现有前端模式和接口封装
3. 实现最小改动并保持可读性
4. 验证交互流程和错误场景

## 触发关键词

前端、UI、页面、交互、表单、组件

## 优先级

中等，若涉及关键流程或权限展示则提高优先级。

## 默认输出要求

说明页面或组件变更点、接口影响和验证方式。

## 适用对象

前端开发者和需要联调前后端的开发者。

## 避免事项

不要把业务规则和权限判断全部写在页面层。

## 示例提示词

为当前页面增加一个新交互流，并保持与现有 UI 组件模式一致。

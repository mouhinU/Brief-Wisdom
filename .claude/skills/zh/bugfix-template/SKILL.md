# Bug 修复模板 Skill

## 作用

当你需要在 Brief-Wisdom 中以可复现、低风险的方式修复缺陷时使用。

## 何时使用

适用于运行时错误、行为不符合预期、回归问题、空值问题、权限缺陷、接口失败或界面不一致等场景。

## 推荐流程

1. 复现问题，并确认具体场景。
2. 跟踪受影响模块，定位根因。
3. 判断问题是来自校验、服务逻辑、持久化、权限还是 UI 集成。
4. 采用最小变更修复根因，而不是只掩盖现象。
5. 尽可能补充回归测试。
6. 用相关测试、构建或手动检查验证修复结果。

## 核心规则

- 不要在不了解根因的情况下只做表面修补。
- 优先聚焦小改动，避免大范围重构。
- 保持未受影响路径原有行为不变。
- 当问题由缺少校验或边界条件导致时，增加防御性检查。
- 如果涉及特殊边界场景，补充说明以便后续维护。

## 触发关键词

bug 修复、回归、缺陷、运行时错误、空指针、失败流程

## 优先级

对用户可见功能、权限或数据完整性问题优先级高。

## 默认输出要求

说明根因、处理思路、涉及文件以及验证结果。

## 适用对象

负责缺陷修复、回归问题和稳定性问题的开发者。

## 避免事项

不要在修复 bug 时引入猜测性改动或无关重构。

## 示例提示词

调查登录失败问题，找出根因后用最小且安全的方式完成修复，并给出验证结果。

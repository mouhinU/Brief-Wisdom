# 上线前检查模板 Skill

## 作用

当你需要在上线或部署前确认变更已经准备就绪、可安全发布时使用。

## 何时使用

适用于合并前检查、部署前确认、交付前验收或影响用户流程、权限、数据或基础设施的变更。

## 推荐流程

1. 回顾变更范围并确认受影响模块。
2. 检查测试、校验和安全边界是否已经完成。
3. 确认配置变更、环境变量和迁移步骤已经记录清楚。
4. 验证主要正常流程和关键边界场景。
5. 准备简短的发布说明和回滚说明。

## 核心规则

- 在主要验证步骤完成前，不要把变更视为已准备好。
- 确认敏感变更具备适当权限控制和审计性。
- 在发布前评估部署与回滚影响。
- 保持上线前检查具体且可执行。

## 触发关键词

上线前检查、发布检查、部署就绪、上线验证、发布验证

## 优先级

对生产可见、权限敏感或影响数据的变更为高优先级。

## 默认输出要求

给出上线前评估、已完成检查项、剩余风险和发布说明。

## 适用对象

负责准备上线或部署的开发者、维护者和发布人员。

## 避免事项

不要跳过对安全、数据、配置和回滚的验证。

## 示例提示词

对这个功能执行上线前检查，并总结其准备情况、风险和发布说明。

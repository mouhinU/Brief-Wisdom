# 测试补全模板 Skill

## 作用

当你需要为 Brief-Wisdom 的变更补充或完善测试时使用。

## 何时使用

适用于缺少测试覆盖、回归测试、服务层测试、接口校验测试或界面交互测试。

## 推荐流程

1. 确认变更行为和风险点。
2. 选择最小且合适的测试粒度：单元测试、集成测试或界面测试。
3. 覆盖正常路径、关键边界情况和失败场景。
4. 保持测试独立、可重复，并与现有测试模式一致。
5. 运行相关测试并确认新增覆盖有实际意义。

## 核心规则

- 优先选择针对性强的测试，而不是过于宽泛且脆弱的测试。
- 测试真实行为，而不是实现细节。
- 优先覆盖最可能出现回归的场景。
- 测试数据和准备步骤保持简洁且可读。
- 优先复用现有测试工具和约定。

## 触发关键词

测试、单元测试、集成测试、回归测试、覆盖率、测试补全

## 优先级

对日常改动为中等；对核心流程或安全敏感行为为高优先级。

## 默认输出要求

说明被测试的行为范围、新增的测试用例以及验证结果。

## 适用对象

为后端、前端或集成变更补充测试的开发者。

## 避免事项

不要添加只验证实现细节、跳过真实失败路径的浅层测试。

## 示例提示词

为新的简历保存流程补充回归测试，并验证主要成功与失败场景。

# 用户系统 Skill

## 作用

用于用户管理、菜单管理、角色管理、登录流程以及系统级账户相关功能。

## 何时使用

当任务涉及账户变更、角色或菜单更新、登录体验调整或管理员功能时使用。

## 核心规则

- 用户与账户逻辑尽量保持在 system 模块中，并复用现有认证链路
- 在修改账户、角色或菜单数据前先完成输入与权限边界校验
- 优先复用现有 DTO 与响应模式，而不是引入新模式
- 角色和菜单变更属于安全敏感操作

## 推荐流程

1. 先查看现有 system 模块流程
2. 复用现有权限与会话模式
3. 通过明确校验和可追踪性完成实现
4. 验证角色与访问行为

## 触发关键词

用户系统、角色、菜单、账号、登录、会话、管理员

## 优先级

角色、权限或登录变更为高优先级；常规账户管理优化为中等。

## 默认输出要求

说明受影响的账号或管理员流程、权限影响以及验证场景。

## 适用对象

负责 system 模块中用户、角色、菜单或账户管理功能的开发者。

## 避免事项

不要创建绕过现有角色或会话约定的账户管理流程。

## 示例提示词

实现一个新的用户账户操作，并确保其遵循当前 system 模块的权限模型。

# DevOps 与部署 Skill

## 作用

用于 Docker、Maven 构建、环境配置、部署行为和运行时相关变更。

## 何时使用

当任务涉及 Docker 配置变更、Maven 构建更新、环境变量、部署步骤或运行时设置调整时使用。

## 核心规则

- 通过环境变量或 profile 配置来外部化配置
- 避免在源码中硬编码密钥、本地路径或环境相关值
- 保持与现有 Docker 与 compose 配置兼容
- 优先采用增量且可回滚的部署修改

## 推荐流程

1. 先确认是否需要新增环境变量或 profile
2. 保持变更向后兼容且对环境安全
3. 仅在必要时更新 Docker 或运行时配置
4. 验证构建、启动和基础运行情况

## 触发关键词

部署、Docker、compose、Maven、环境变量、运行时配置

## 优先级

影响部署的变更为高优先级；本地配置一致性更新为中等。

## 默认输出要求

说明变更的环境或部署面、配置影响，以及验证命令或启动检查。

## 适用对象

负责部署、Docker、环境配置或构建行为的开发者与运维人员。

## 避免事项

不要硬编码运行时值，也不要破坏现有容器或 Maven 工作流兼容性。

## 示例提示词

更新部署配置，使其与当前的 Docker 和 Maven 工作流保持兼容。

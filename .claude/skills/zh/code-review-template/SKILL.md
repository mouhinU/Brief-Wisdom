# 代码评审模板 Skill

## 作用

当你需要对 Brief-Wisdom 的变更进行正确性、可维护性、安全性和一致性评审时使用。

## 何时使用

适用于 PR 评审、变更检查、架构审查或合并前质量检查。

## 推荐评审流程

1. 阅读变更范围，理解预期行为。
2. 检查实现是否符合项目架构和模块边界。
3. 评估逻辑正确性、边界情况、异常处理和校验。
4. 查找安全问题、权限缺口或敏感数据处理风险。
5. 评估测试覆盖是否足够，以及回归风险是否可控。
6. 用明确的严重级别和可执行建议总结结论。

## 评审关注点

- 逻辑正确性与流程完整性
- 命名、结构和可读性
- 对现有模式和服务的复用情况
- 安全性与权限边界
- 测试覆盖与回归风险
- 性能与可维护性

## 触发关键词

代码评审、PR 评审、评审清单、质量检查、变更检查

## 优先级

对于影响安全、数据完整性、共享服务或核心流程的变更，优先级较高。

## 默认输出要求

给出简洁的评审摘要，按严重程度列出问题，并提供建议修复或后续动作。

## 适用对象

参与 PR 或代码变更评审的开发者与维护者。

## 避免事项

不要在存在安全边界缺失、缺少校验或引入不必要复杂度的情况下批准变更。

## 示例提示词

请评审这个补丁的正确性、架构一致性以及潜在的安全和回归风险。
